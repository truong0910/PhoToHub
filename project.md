🛠️ CHI TIẾT DỰ ÁN: PHOTOHUB (ECOSYSTEM FOR PHOTOGRAPHERS & EQUIPMENT RENTAL)1. KIẾN TRÚC HỆ THỐNG (SYSTEM ARCHITECTURE)Dự án được xây dựng theo mô hình Serverless Architecture (BaaS) dựa trên nền tảng Supabase.  Frontend Web (React.js + TailwindCSS): Dành cho Studio Manager (Admin) quản lý kho bãi, nhân sự và theo dõi dòng tiền.  Frontend Mobile (Flutter + BLoC/Provider): Dành cho Khách hàng lướt xem portfolio, đặt lịch, thuê máy và nhận cập nhật realtime.  Backend (Supabase Serverless Cloud): Chịu trách nhiệm lưu trữ PostgreSQL, xác thực JWT, kích hoạt cổng lắng nghe dữ liệu Realtime, lưu trữ file Media và chạy các logic kiểm tra thông qua Edge Functions.  2. THIẾT KẾ CƠ SỞ DỮ LIỆU (POSTGRESQL SCHEMAS TRÊN SUPABASE)Bạn cần tạo các bảng chính sau đây trong Supabase Table Editor với các ràng buộc chặt chẽ:  Bảng profiles (Kế thừa từ auth.users của Supabase):  id (uuid, Primary Key, Foreign Key kết nối sang hệ thống Auth).  full_name (text), phone (text), avatar_url (text).  role (text: có 3 giá trị client, photographer, admin).  Bảng equipment (Quản lý kho máy ảnh, ống kính):  id (uuid, Primary Key).name (text - Ví dụ: Nikon D700, Canon 700D, Lens 85mm F1.4).category (text: body, lens, lighting).price_per_day (numeric - Giá thuê theo ngày).image_url (text - Link ảnh lưu trên Supabase Storage).  status (text: available - sẵn sàng, maintenance - đang bảo trì).Bảng bookings (Quản lý lịch chụp và đơn thuê máy):  id (uuid, Primary Key).client_id (uuid, Foreign Key liên kết bảng profiles).  photographer_id (uuid, Foreign Key liên kết bảng profiles, có thể rỗng nếu khách chỉ thuê máy).  equipment_id (uuid, Foreign Key liên kết bảng equipment, có thể rỗng nếu khách chỉ thuê thợ).  start_date (timestamp - Ngày bắt đầu chụp/thuê).end_date (timestamp - Ngày kết thúc).status (text: pending, approved, ongoing, completed, cancelled).  total_price (numeric - Tổng giá trị đơn hàng).3. KỊCH BẢN TÍNH NĂNG CHI TIẾT (USER STORIES & FLOWS)🔹 Luồng 1: Xác thực & Phân quyền (Supabase Auth)Cách chạy: Khi người dùng đăng ký tài khoản trên App Mobile hoặc Web Admin, Supabase Auth sẽ tự động cấp một mã định danh UUID bảo mật. Bạn viết một Database Trigger trên Supabase: mỗi khi có user mới ở bảng Auth, nó sẽ tự động chèn một dòng tương ứng vào bảng profiles với quyền mặc định là client.  Khi phỏng vấn: Nhấn mạnh việc bạn dùng chính sách Row Level Security (RLS) của PostgreSQL trên Supabase để chặn không cho tài khoản client được quyền sửa đổi trạng thái thiết bị trong bảng equipment (chỉ admin mới có quyền này).🔹 Luồng 2: Xem Portfolio & Đặt lịch thông minh (Flutter Mobile App)UI/UX: Giao diện Flutter hiển thị danh sách các album ảnh độ phân giải cao được tối ưu hóa tải nhanh nhờ Supabase Storage CDN.  Logic Đặt lịch: Khách hàng chọn một chiếc máy (ví dụ: Nikon D700) và chọn ngày chụp. Trước khi gửi yêu cầu, ứng dụng sẽ gọi một bộ lọc truy vấn dữ liệu (Query) vào bảng bookings để kiểm tra:  Điều kiện: Liệu chiếc máy đó đã có đơn hàng nào ở trạng thái approved nằm trong khoảng ngày khách chọn hay chưa? Nếu có, hệ thống sẽ báo đỏ và khóa nút đặt lịch để tránh bị trùng lặp thiết bị.  🔹 Luồng 3: Đồng bộ trạng thái Realtime giữa Web và MobileBối cảnh: Khách hàng gửi đơn đặt lịch chụp ở trạng thái pending.  Hành động trên Web (React.js): Admin mở Dashboard, thấy đơn hàng mới hiện lên, bấm nút "Duyệt đơn" (Approved). Hệ thống cập nhật cột status trong bảng bookings thành approved.  Phản hồi trên Mobile (Flutter): Nhờ tính năng Supabase Realtime Stream, ứng dụng di động của Khách hàng lập tức lắng nghe thấy sự thay đổi của dòng dữ liệu đó, tự động đổi màu trạng thái từ vàng (Pending) sang xanh lá (Approved) và hiển thị thông báo chúc mừng mà không cần tải lại trang.  🔹 Luồng 4: Quản lý Kho & Báo cáo số liệu (React.js Web Admin)Quản lý tài sản: Admin thực hiện CRUD thiết bị, theo dõi thiết bị nào đang cho thuê ở ngoài, thiết bị nào đang hỏng cần bảo trì.  Biểu đồ phân tích (Analytics): Sử dụng thư viện Recharts/Chart.js trên React.js để đọc dữ liệu từ bảng bookings đã hoàn thành (completed), tính tổng doanh thu theo tháng và vẽ biểu đồ cột.
